# GPU Inference Platform — CLAUDE.md

## Project purpose

Production-style GPU inference platform for NVIDIA AI infrastructure.
Stack: PyTorch, ONNX, TensorRT, NVIDIA Triton, FastAPI, C++/CUDA, Kubernetes, Prometheus/Grafana.

## Rules

1. Prefer simple, reproducible scripts over complex abstractions.
2. Every major component must have tests or a validation script.
3. Never invent benchmark numbers. All results must be generated by scripts in `benchmarks/`.
4. Keep the README interview-ready.
5. Use Python 3.11.
6. Use Makefile targets for common workflows.
7. Commands that require a GPU must be clearly marked `[GPU REQUIRED]`.
8. CPU-only fallback tests must exist for CI (no GPU required in `make test`).

## Conventions

- FastAPI app lives in `app/`
- Model export/build scripts live in `models/`
- Kubernetes manifests live in `k8s/`
- Benchmark scripts write JSON/CSV to `benchmarks/results/`
- Profiling commands are documented in `profiling/*.md`; saved reports go in `profiling/reports/`
- Shell scripts in `scripts/` are thin wrappers around Python/Docker/kubectl commands

## GPU-required components

- `make build-tensorrt` — needs CUDA + TensorRT
- `make run-triton` — needs CUDA + Docker + NVIDIA Container Toolkit
- `make build-cuda` — needs CUDA + nvcc
- Docker Compose Triton service — needs NVIDIA Container Toolkit

## Testing

- `make test` = `pytest -m "not gpu and not triton"` — runs on CI without GPU
- Mark GPU tests with `@pytest.mark.gpu`
- Mark Triton-dependent tests with `@pytest.mark.triton`
